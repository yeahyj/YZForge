# 07. 生成文件

## 哪些文件是生成的

常见生成文件：

```text
assets/app/registry/**/*.generated.ts
assets/app/contracts/**/*.generated.ts
assets/app/bootstrap/install.generated.ts
assets/modules/<Module>/code/generated/entry.ts
assets/modules/<Module>/code/generated/assets.ts
assets/modules/<Module>/code/generated/config.ts
assets/modules/<Module>/code/generated/content-packs.ts
assets/libraries/<Library>/code/generated/entry.ts
assets/libraries/<Library>/code/generated/assets.ts
assets/libraries/<Library>/code/generated/config.ts
assets/content-packs/<Owner>/<Pack>/manifest.generated.json
```

`code/generated/*.ts` 是现在推荐的生成目录。不要再新增 `assets.generated.ts`、`entry.generated.ts` 这类旧式平铺文件。

## 判断标准

生成文件顶部会有类似：

```ts
// <auto-generated>
// generator: yzforge@...
// source: ...
// hash: ...
// </auto-generated>
```

看到这个头，就不要手改。

## 修改方式

想改变生成结果，改源头：

| 你想改 | 应该改哪里 |
| --- | --- |
| Module 名称、bundle、依赖 | `module.json` |
| Library 名称、bundle、依赖 | `library.json` |
| 公开参数或 token 类型 | `code/public.ts` |
| 资源清单 | `res/` 下资源和 prefab |
| UI refs | Cocos prefab 节点和组件 |
| ContentPack 元信息 | `content-pack.json` |
| 配置表入口 | `res/content/config/manifest.json` 和配置源 |

然后运行：

```bash
npm run yzforge:generate
```

## 检查生成物是否最新

```bash
npm run yzforge:generate:check
```

如果失败，说明生成物不是当前源文件的结果。直接运行：

```bash
npm run yzforge:generate
```

再看 git diff。

## 清理生成物

先预览：

```bash
npm run yzforge:clean:generated:check
```

确认后清理：

```bash
npm run yzforge:clean:generated
npm run yzforge:generate
```

清理生成物会删除带 auto-generated 头的 TS 和 generated JSON。不要用它清理手写代码。

## 常见误区

- 不要手动修 `hash`。
- 不要为了过 TypeScript 去改 `generated/entry.ts`。
- 不要从其他 Scope import 对方的 `generated/assets.ts`。
- 不要把业务逻辑写进 `generated/entry.ts`。
- 不要把 `code/generated/` 当成手写扩展点。

生成器是单向入口：源文件到生成物。修问题永远优先修源文件。
